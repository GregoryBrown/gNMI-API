# gNMI-API
Simple API for connecting to an gNMI network device and upload to an Elasticsearch Database

## Requirements
* Python3.6+
* Elasticsearch TSDB running 7.x+ (If you want to upload to ES)
* Ability to do pip install

## Installing in a virutal environment

```python
lab@web-ott-execution-server-1:~/test$ python3.7 -m venv venv
lab@web-ott-execution-server-1:~/test$ source venv/bin/activate
```

## Installing the keys plugin for pyang if you want to use Get on operational data

### Getting the yang files off the box 

The most common way to get yang files off a device is by getting the schemas via Netconf-Yang, then interate over them doing a get-schema on each yang model and save those responses locally as yang files.

### Generating the keys file from the yang models

First locate where your version of pyang is installed at
```python 
(venv) lab@web-ott-execution-server-1:~/dev/gNMI-API$ which pyang
/home/lab/dev/gNMI-API/venv/bin/pyang
```
Copy the keys_plugin.py file to pyang's lib directory
```python
(venv) lab@web-ott-execution-server-1:~/dev/gNMI-API$ cp keys_plugin.py  venv/lib/python3.7/site-packages/pyang/plugins/
```
Run pyang with the keys option on the yang file(s) you want to generate keys for, and copy the output to a file to be used in the API
